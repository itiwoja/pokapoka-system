**用途:** 顧客用タブレットシステム（自前Web注文アプリ）の客席端末
**必要台数:** 16台
**当初の選定軸:** 10インチ前後 / RAM4GB / USB-C / Wi-Fiモデル / 注文画面・動画再生対応 / 本体¥2万以下

## 結論（最重要）

1. **第一候補を旧モデル Lenovo Tab K10 → 現行 Lenovo Tab M11 / B11 に差し替え**
   - K10 は2021年世代で、16台同一ロットの新品確保が困難なため。
2. **キオスク化は高価なMDM不要。Fully Kiosk Browser で安く確定できる**
   - 自前Web注文アプリのURLを起動時に全画面固定し、他アプリへの離脱を禁止できる。どのメーカーの安価なAndroidでも動作するため、廉価機も候補に残せる。
   - 無料版で基本のURL固定は可能。買い切りPLUSライセンス（1台あたり数百〜¥1,500程度）で機能が安定。
3. **「本体¥2万以下」はクリア可能だが、ケース・フィルム・キオスク込みの実質コストは1台¥2万を少し超える。**
   - 予算は **1台あたり約¥2.2万・16台で約36万円** で見ておくと安全。

**Why:** 限られた予算で16台を確実に調達し、客席で安定運用するため。
**How to apply:** 機器提案（P1タスク）では K10 ではなく現行Lenovo M11/B11 を第一候補に。キオスク化は Fully Kiosk Browser 前提で見積もる。

## 設計に効く重大な示唆（要件に反映すべき）

1. **動画はローカル保存ループが鉄則（F-01直結）。** Wi-Fiの真のボトルネックは同時接続数ではなく「16台が動画を流す帯域＝WAN回線」。スクリーンセーバー動画は端末内に置けば回線負荷ほぼゼロ → 性能要件9.1の達成可否を左右。
2. **常時給電のバッテリー膨張対策が必須。** 民生機の通電しっぱなしはメーカー保証外。充電上限80%設定＋タイマー充電＋放熱で運用カバー。安全最優先なら業務用の常時給電対応機（オーディーエス TA2C-CS8）もあるが予算外。
3. **オフライン耐性。** Wi-Fi瞬断で注文消失が内製事例の最大の地雷。一時保持→復帰時送信を実装に織り込む（プリンタ連携の印字フォーマットも要注意点）。
4. **非接触（マグネット/Qi）充電**はコネクタ摩耗・水濡れを回避でき、ぐるなびFO等も採用。据置席で検討価値あり。
