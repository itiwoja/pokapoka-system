# レシートプリンターのネットワーク接続検討（2026-07-17 チャット整理）

> Claude Codeとのチャットで進めた、実機印刷（PR #166 / #144）用にレシートプリンターをネットワークへ繋ぐ検討のログ。
> 関連: [実機印刷テスト手順 (2026-07-17)](../docs/実機印刷テスト手順_2026-07-17.md)

## 背景：なぜネットワーク接続が要るか

目的は「KDS端末で伝票を発行したら、実際に紙のレシートが出てくる」こと。

- PR #144 / #166（コミット `fe67391`）で `POST /api/print` を追加。KDSの「プリンター設定」でプリンターのIPを登録すると、relay-serverがそのIP宛にポート9100（ESC/POS RAWポートの事実上の標準）へ**生TCPソケットで直接ESC/POSコマンドを送信**して印字させる（`relay-server/printer.js`）
- IP未設定の場合は従来どおりブラウザの `window.print()`（印刷ダイアログ）にフォールバックする
- KDS(ブラウザ)は生TCPソケットを開けないため、relay-server(Node)が仲介する設計

**Why:** ブラウザから直接プリンターへ生TCP送信はできない制約のため、relay-server経由の設計になっている。
**How to apply:** 自動印刷（ダイアログなしで紙が出る）を実現するには、**プリンターが店内LAN上でIPアドレスを持っていること**が必須。無線(WiFi)である必然性はなく、有線LANケーブル直結でも目的は達成できる。

## プリンター機種と接続インターフェースの制約

対象機種は **Star mC-Print3 (MCP31LB WT JP)**（`relay-server/printer.js` コメントに記載）。

- 標準搭載インターフェース: **Ethernet(LAN)・USB・Bluetooth・Lightning**
- **WiFi(無線LAN)は標準非搭載**。無線接続するには別売りの **「MCW10 Wireless LAN Unit」アダプター** が必要（Star公式サイトで裏どり済み）

**Why:** 「WiFiに繋ぐ」と言っても、この機種は無線チップを内蔵していないため、アダプターの有無で取れる手段が変わる。
**How to apply:**
- MCW10アダプターが**ある**場合 → Star公式アプリ「Star Quick Setup Utility」(iOS/Android/Windows)で無線LAN設定が可能
- MCW10アダプターが**ない**場合 → 無線では繋がらない。有線LANケーブルでプリンター⇔ルーター(またはPC直結)を繋ぐのが現実解

### MCW10アダプターの有無の確認方法

プリンターの**FEEDボタンを押しながら電源ON**すると自己診断シート(テスト印字)が出る。この印字内容に

- 「**WLAN**」の項目（SSID・電波強度など）が**ある** → アダプター付き。無線接続できる
- 「**LAN**」(Ethernet)の項目**しかない** → アダプターなし。無線は不可

で判別できる。IPアドレスもこの同じ印字の「IP Address」欄に記載される（`docs/実機印刷テスト手順_2026-07-17.md` の手順と同一）。

**未解決（持ち越し）:** 2026-07-17時点、実際のテスト印字にWLAN項目があるかどうか未確認。次回、印字内容の写真または項目名の書き起こしで判定する。

## 検討した代替案：スマホのテザリング

店舗ルーターがまだ使えない/未確定の場面向けに、**スマホのWiFiテザリング(ホットスポット)を店舗ルーターの代わりのネットワークとして使う**案が出た。

**Why:** 実機検証を店舗ルーター調達前に始めたい場合、手元のスマホでその場にネットワークを作れる。
**How to apply:** テザリングでPC/iPadを繋ぐこと自体は問題ないが、**プリンター側がテザリングを無線で掴めるかどうかは上記のMCW10アダプター有無と同じ制約を受ける**（店舗ルーターでもスマホのホットスポットでも、プリンター側に無線チップが無ければ同じく無線接続不可）。アダプターが無ければ、テザリングしてもプリンターは有線LAN直結が必要になる。
