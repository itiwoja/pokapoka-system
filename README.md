# pokapoka-system

沖縄みらいAI＆IT専門学校が土鍋飯ぽかぽか様のオーダーシステムを制作するためのリポジトリ。

> **本リポジトリのスコープは「厨房端末（KDS）画面」と、それに予約を流し込む「店内中継サーバー」**です。客席の注文端末（各卓のタブレット）は別チームが制作し、管理画面・予約サイト側などは対象外です。

## 構成

| パス | 内容 |
|---|---|
| `kds-a-grid.html` | **KDS（デシャップ）画面本体**。単一HTMLで自己完結 |
| `relay-server/` | **店内中継サーバー**。TableCheck 予約の取込と KDS の Web 配信（[詳細 README](relay-server/README.md)） |
| `config/config.example.json` | 端末接続先などの設定雛形（実体 `config/config.json` は .gitignore 済み） |
| `serve-log.js` / `serve-log.test.js` | 提供時間計測ロジックの Node テスト用参照実装 |
| `docs/` | PRD・要件定義・DB設計などの設計ドキュメント |
| `knowledge/` | 打合せ議事録・調査メモ・裏どり結果 |

## KDS（厨房オーダー表示システム）

**`kds-a-grid.html`** が本線の KDS 画面です。HTML / CSS / JavaScript のみの**単一ファイルで自己完結**（フレームワーク・外部CDN不使用）し、オフライン環境のタブレット（Android / iPad）・13.3" IP65 産業用タッチパネルで動作します。**産業用計器 UI**（商用デシャップ Okage DDS を参考にした、黒ヘッダー＋明るい盤面・色は「意味」にのみ使う設計）で、カードグリッド＋予約ストックの2ペイン構成。緊急度＝カード上帯（緑→琥珀→赤）、種別＝**カード背景のトーン差（新規=淡青／予約=淡金）**＋アンダーライン/チップ、赤は「10分超の経過警告」専用。

主な機能:

- 品目タップ完了・全完了カードの自動消去
- コンロ番号（1〜10）の割当とタイマーロック。**同一コンロ番号は同時に1卓のみ**（物理コンロは1台のため、使用中は他カードの同番号をロック）
- 予約ストック→進行中への配膳移動（卓番は誤入力防止の**ダイヤルUI**で選択）
- キッチン/ホール権限モード、チビ伝の自動印刷
- 経過時間による色アラート、**10分超の点滅＋音声アラート**、新着通知音（別トグル）
- **グリッド/レーン表示切替（カテゴリ別）**
- LocalStorage + BroadcastChannel による2タブ同期

デモ／メニューデータは実メニュー（山城牛の土鍋御膳／単品／ドリンク）を使用しています。

### 動作確認

```sh
# ① KDS 単体で開く（file:// では動作しないブラウザがあるため）
python -m http.server 8000
# → http://127.0.0.1:8000/kds-a-grid.html

# ② 中継サーバー経由で開く（TableCheck 予約取込のブリッジが自動注入される）
node relay-server/server.js
# → http://127.0.0.1:8000/       … KDS（デシャップ）
# → http://127.0.0.1:8000/demo   … 予約デモコンソール
```

`window.KDS_ORDERS` に注文データ（`{id, table, type, start, people, items:[{name, qty, options, allergies, done}]}`）を投入すると表示されます。アレルギー情報は `allergies`（`allergy` / `allergyInfo`・配列も可）で受け取り、品目直下に「アレルギー: …」として常時表示します。空の場合は「デモデータを投入」ボタンでサンプルを表示できます（中継サーバー配信時は自動デモを抑止）。

## TableCheck 予約連携（店内中継サーバー）

`relay-server/` の店内中継サーバーが TableCheck から予約（メニュー・人数等）を **30秒間隔の差分 pull**＋**15分ごとの当日全件リシンク**で取得し、KDS の予約ストックへ流し込みます。**依存パッケージゼロ・Node 18+ のみ**で動作します。

```
【クラウド】               【店内ミニPC = 中継サーバー】          【KDS端末】
TableCheck ◀── 外向きpull ── server.js ── /api/stock(JSON) ──▶ kds-bridge.js → 予約ストックへ
```

- API 契約前でも**モックモード**で動作し、`/demo` の予約デモコンソールから予約の作成・人数変更・キャンセル → KDS 反映までを実機で確認できる
- デシャップモニター・注文端末の接続先（host/port）は **`config/config.json` に集約**（雛形: `config/config.example.json`。優先順位は 既定値 < config.json < 環境変数。**API キーは環境変数のみ**）
- 起動方法・エンドポイント・LAN 配信時の注意（固定IP・ファイアウォール）は [relay-server/README.md](relay-server/README.md) を参照

### データ保存方針: 何も保存しない

打合せ確定（2026-06-18 議事録）に基づき、**注文・予約はサーバーに保存しない**。中継サーバーは**当日分の予約のみメモリ保持**し、再起動時は TableCheck から当日全件を取り直します（初回リシンク成功まで `/api/stock` は 503 を返し、KDS 側は直前表示を保持）。永続 DB 設計は将来構想として `docs/db-design.md` に保留しています。

## 提供時間の自動計測（Issue #29）

注文受付（`start`）から**全品目完了までの提供時間を自動記録**し、LocalStorage（キー `kds_serve_log_v1`）へ追記保存します。PRD の KPI「平均提供時間 MVP前比20%減」の**導入前ベースラインデータ**として用います。

- 全品目をタップ完了した瞬間に、完了時刻・提供時間（`serveMs`）を記録
- `orderId` で重複ガード（2タブ同期・再完了での二重計上を防止）
- ヘッダーの表示ボタンは削除済み（#112）。参照・エクスポートは下記のコンソールアクセサから行う

計測ロジックは `kds-a-grid.html` にインライン実装（単一HTML要件のため）していますが、同一ロジックを `serve-log.js` に切り出して Node テストで検証しています。

```sh
node serve-log.test.js
```

コンソールからの参照用アクセサ:

```js
window.KDS_getServeLog()    // 記録レコード配列
window.KDS_getServeStats()  // 件数・平均・最大・最小・10分超件数
window.KDS_exportServeCSV() // CSV ダウンロード
window.KDS_clearServeLog()  // ログ全消去
```

> **注**: `serve-log.js` は Node テスト用の参照実装です。`kds-a-grid.html` 内のインライン版とロジックは同一なので、片方を変更したら両方（と `serve-log.test.js`）を同期してください。

## 遅延アラート：10分超の点滅＋音声（Issue #12）

厨房が「見ていない瞬間」の提供遅延に気付けるよう、経過10分超のカードを強調します。

- **点滅**: 10分超カードに赤いアウトラインを付与し明滅（`prefers-reduced-motion` 時は静的な赤枠）
- **音声**: 10分を跨いだ瞬間にビープ（WebAudio・外部ファイル不要）。継続中は一定間隔で再アラート
- ヘッダーの **「🔔 アラート音」** トグルで ON/OFF（LocalStorage に永続、デフォルト ON）
- タイマーロック中のカードは再アラートしない（＝確認済み扱い）

> ブラウザの自動再生制限により、音は初回のユーザー操作（タップ等）以降に鳴ります。点滅は常に表示されます。

## レーン表示：カテゴリ別分割（Issue #13）

ヘッダーの **「グリッド／レーン」** 切替で、注文カードのグリッドとは別に、**未完了品目をカテゴリ別3レーンに横断表示**できます。コンロ側・中間スタッフ側での作業分担が目的です。

- 3レーン: **🍚 ご飯類**（土鍋御膳・ご飯もの）／**🍢 サイド**（単品）／**🥤 ドリンク**
- 各レーンに未完了品目を **[卓番] 品名 ×数量（＋オプション）** で並べ、経過時間レベルで左枠を色分け
- 品目タップで完了（`doneMap` を共有）→ 完了品目はレーンから消える。グリッドと状態・2タブ同期は共通
- 表示モードは LocalStorage に永続

> 品名→カテゴリは POS データに無いため、KDS 内の分類関数（`itemCategory`）で自動判定します（実メニュー基準・未知はサイドへ）。POS のメニューが増えたら分類キーワードの追記を検討してください。
