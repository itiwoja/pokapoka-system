# デシャップ画面 仕様（会議決定の統合・実装用）

> 出典：docs/PRD.md ＋ knowledge/（2026-05-29 KDS画面詳細要件 / 2026-06-03 色分け・座席DB / 2026-06-04 議事録 / 2026-06-18 議事録）の確定事項を統合。
> 目的：厨房の「デシャップ（dish-up）画面」を**とりあえず完成レベル**まで。POS/テーブルチェック未接続のためデモデータで動作。

## 技術制約（厳守）
- **単一HTMLファイル**・HTML/CSS/Vanilla JS(ES2020)のみ・**外部CDN/参照ゼロ**・**完全オフライン**。
- **産業用計器UI（完全機能特化）**：商用デシャップ Okage DDS を参考に「黒ヘッダー `#1A1A1A` ＋明るい盤面 `#E9ECEF`」でクロームと作業面を分離。白カード `#FFFFFF` に清潔な実影＋角丸で"意図された計器"に。文字主 `#14161A` / 副 `#565B61`。厨房の距離から一瞬で状態を読ませることを最優先し、色は「意味」にのみ使う（緊急度=カード上帯の高彩度塗り 緑→黄→赤 / 種別=アンダーライン・チップ 新規=藍/予約=金、赤は経過警告10分+専用）。主要CTAはオレンジ `#C25200`（赤と分離）。数量は黒塗り四角＋白数字、コンロは既定"使用中のみ"の細帯＋「割当」で1〜10展開。過度な装飾は用いない。全要素AA（本文4.5:1/非テキスト3:1）・タップ域（品目64px+/補助44px+）準拠。
- **データ契約**：`window.KDS_ORDERS=[{id,table,type:"new"|"reserved",start,people,items:[{name,qty,options,allergies,done}]}]` を1秒ポーリング。
- **同期＝BroadcastChannel＋LocalStorage**：**同一端末内の複数タブ／ウィンドウ間のみ**同期する。BroadcastChannel の仕様上、**別端末（別タブレット）間は同期しない＝既知の制約（#70）**。単一HTML・外部参照ゼロ・完全オフラインの制約下では別端末間同期は非対応とし、必要になった場合はサーバ導入を伴う別 Intent として扱う。**データ保存は当日分のみ・永続DB無し**（議事録6/18）。
- AA(本文4.5:1/非テキスト3:1)・卓番号28px+・品名22px+・1MB以下。
- **タップ領域**：主要操作（品目タップ）は**64px+**。補助ボタン（コンロ番号・伝票プレビュー・権限切替等）は**WCAG AA相当の44px+**を下限とする。コンロは5列×2行の画面幅制約があるため一律64pxは課さない。
- **日本語オンリー**（オーナー決定）。多言語対応・言語トグルは本リポジトリ（厨房KDS）のスコープ外。

## 画面レイアウト（2ペイン）
- **左＝メイン配膳エリア**：進行中オーダーのカードグリッド。**縦スクロールの折返しグリッド・時系列順（新着は末尾＝行優先で追加＝配膳/FIFO順）**。先頭挿入はしない＝作業中カードの位置を動かさない。新着の気づきを担保していた「新着音」トグルは不要と判断し**廃止**（10分超の遅延アラート音も同様に廃止＝音声通知なし）。
- **右＝予約ストックパネル**：当日予約を時間順に縦並び（縦スクロール）。
- **権限モード切替**（ヘッダ）：**キッチン**＝全操作可／**ホール**＝読取専用＋予約入力のみ。

## 注文端末→デシャップ送信項目

- 注文端末側の入力順：**卓番（最初に入力）** → **予約 or 新規** → **メニュー選択（スライド式でメニュー画像をタップ）** → **数量（＋／－ボタンで増減）** → **オプション（アレルギー表示あり、オプションあり／なし選択）**。
- デシャップ行き必須項目：**卓番**、**注文したメニュー名**、**数量**、**オプション内容（アレルギー情報含む）**。
- 予約/新規区分は `type:"reserved"|"new"` として受け取り、カード色とバッジに反映する。
- アレルギー情報は `allergies` 推奨。注文端末側の都合で `options` に含める場合も、品目直下に常時表示する。

## オーダーカード（メイン）
- **色（最新決定）**：新規=**水色系(淡)** / 予約=**黄色系(淡)** / **赤は経過警告(10分+)専用**（カード背景に赤を使わない）。予約黄色と経過琥珀が混同しないよう低彩度カード＋高彩度バッジ＋数値併記。
- 表示：卓番号(大)・人数(可能なら大人/子供内訳)・受付HH:MM・**経過時間(分・3段階 0-4緑/5-9琥珀/10+赤)**・品名(略称対応/長文は省略)・数量・**オプション/アレルギー/備考は常時表示**（品目直下・副色）。
- **品目タップ**＝打ち消し線＋done切替（再タップ取消）。**全品目done→300msフェードで自動消去**。
- **コンロ番号 1〜10**（2行）：タップ循環 **スケルトン→白(コンロ使用中)→赤(タイマー管理中)→リセット**。他カードから同番号押下可（移動対応）。状態はLS＋BC同期（同一端末内・#70）。経過警告の赤と区別（小さめ・番号ラベル付き）。
- **長押し(0.8〜2秒)＋アニメで移動**（並べ替え＝配膳順変更）。誤操作防止に長押し必須＋取消/戻る。
- **チビ伝印刷（簡易）**：**注文単位で全品目を1枚にまとめた伝票**を出す（卓番＋品名が読めるサイズ）。Webでは**伝票レイアウトのプレビュー＋手動 window.print()**で代替。
  - 新着注文で伝票プレビューを自動的に開く「自動プレビュー トグル」（旧称「自動印刷」）は不要と判断し**廃止**。プレビューはカードの印刷ボタンからの手動表示のみ。
- **スコープ外の先行実装（既定OFF）**：**タイマーロック（停/再）**・**注文削除（×）** は会議未決定のためフィーチャーフラグで無効化し、UIに表示しない（`FEATURE_TIMER_LOCK` / `FEATURE_DELETE_ORDER`）。正式決定が出た段階で本仕様に追記のうえ有効化する。

## 予約ストックパネル（右）
- 各カード：予約時間（最大表示）・人数(大人/0-2歳内訳)・名前・メニュー（名称/略称・常時表示）。**時間順に自動ソート**・縦スクロール。
- **追加ボタン(＋)**→フォーム（時間・人数・名前・メニュー[チェック＋個数]）。営業前一括入力＋営業中追加。
- **30分前通知**：予約時間30分前に該当カード右上にバッジ/点滅で通知（自動移動しない）。
- **来店→メインへ**：席番(卓番)を入力して「着手/配膳へ」ボタンでメイン配膳エリアへ移動（予約→進行中）。
- 席だけ予約(メニュー未確定)はストックに載せない（新規扱い）。

## モード/権限
- キッチン：品目完了・コンロ・移動・印刷の全操作。
- ホール：閲覧のみ＋予約ストックの追加/席番入力。

## テーブルチェック連携（検討中）

- **API連携は可能**。ただし追加料金が発生する（金額は担当者間の打ち合わせ後に確定）。
- 取得できるデータの中身・形式は別担当者との打ち合わせが別途必要。
- 予約通知メールの可否も未確認（同打ち合わせで確認予定）。
- **現状：未接続。連携コスト・仕様確定まではデモデータで運用。**

## デモ（POS未接続）
- 起動時/空時に投入：新規・予約混在の進行中オーダー（経過バラけ・オプション/備考有無・done混在・15件規模）＋予約ストック数件（30分前通知が出るものを含む）。
- コンソールから `window.KDS_ORDERS` 書込でも反映。
