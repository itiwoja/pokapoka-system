## Claude が実施する手順

### 1. サーバー起動

1. `git worktree list` で他セッションの状況を確認(このワークツリーで作業する)
2. サーバーが動いているか確認: `curl http://localhost:8000/api/health`
3. 動いていなければバックグラウンドで起動:
   ```
   node relay-server/server.js
   ```
   (config は `host: "auto"` なので、その時のWi-FiのLAN IPを自動検出して待ち受ける)
4. 起動ログの `起動: http://<IP>:8000` からLAN IPを控える
5. `curl http://<IP>:8000/` が 200 を返すことを確認

### 2. PCに3画面を表示する(ブラウザタブ3つ)

Chromeブラウザ(mcp claude-in-chrome)でタブを開く:

| タブ | URL | 用途 |
|---|---|---|
| ① 予約注文画面(デモコンソール) | `http://<IP>:8000/demo` | テスト用の予約・注文を投入する操作卓 |
| ② チビ伝フォーマッター(スタイル設定) | `http://<IP>:8000/slip-style-designer.html` | 伝票の見た目をその場で調整 |
| ③ iPad接続用QR | `http://<IP>:8000/qr` | ユーザーがiPadで読むQRを表示 |

表示したら各タブのスクリーンショットを撮って正常表示を確認する。

### 3. iPadでKDSを開いてもらう

- タブ③のQR画面を前面に出し、ユーザーに「iPadのカメラでKDSのQRを読んでください」と伝える
- iPadはこのPCと**同じWi-Fi**にいること
- 開けないと言われたら: Windowsファイアウォールの許可ダイアログ(初回起動時)で「アクセスを許可する」を押したか確認

### 4. プリンターIPの登録(ユーザー操作・初回だけ)

- KDS画面(iPadまたはPCの `http://<IP>:8000/`)のヘッダー「プリンター設定」を押してもらい、
  プリンター(Star mC-Print3)の店内LAN IPを入力してもらう
- IPが不明な場合: プリンターの**FEEDボタン長押し**で設定情報が印字され、そこに IP Address が出る

### 5. テスト注文を投入する

タブ①(予約注文画面)から予約を作成する。画面操作が難しければ、KDSのタブで
`window.KDS_ORDERS` へ直接投入してもよい(データ契約):

```js
window.KDS_ORDERS = [{
  id: "test-" + Date.now(), table: "A1", type: "new", start: Date.now(),
  items: [
    { name: "特上土鍋ご飯", qty: 1, options: "固め" },
    { name: "だし巻き玉子", qty: 2 }
  ]
}];
```

約1秒でKDSにカードが出る。

### 6. スタイル設定 → 印刷の流れを見せる

1. タブ②(チビ伝フォーマッター)で見た目を変える(例: 卓番XL・数量「2 個」表記)
   — 設定は**サーバー保存**なので、どの端末のKDSにも反映される
2. KDSの注文カードの「伝」ボタンで伝票プレビューを開き、設定が反映されているか確認
3. ユーザーに「印刷」ボタンを押してもらう
4. プリンターから紙が出れば**成功🎉**。出なければ下のトラブル表へ

### 7. テスト後

- 結果(成功/失敗・症状)を確認し、PR #166 のテスト計画チェックリストを更新する
- サーバーはユーザーが続けて使うなら起動したままにする

## トラブルシューティング

| 症状 | 対処 |
|---|---|
| 印刷ボタンでブラウザの印刷画面が出た | 実機に届いていない。①「プリンター設定」のIP入力ミス ②プリンターとPCが同じWi-Fiか ③プリンターの電源・用紙 を順に確認 |
| iPadでQRを読んでも開かない | ①iPadが同じWi-Fiか ②Windowsファイアウォールの許可 ③サーバーが127.0.0.1待ち受けになっていないか(/qr に警告が出る。config/config.json の host を "auto" にして再起動) |
| 見た目が反映されない | 設定ツールを `file://` で開いていないか。必ず `http://<IP>:8000/...` で開く(サーバー保存が効かないため) |
| 文字の大きさがプレビューと違う | 仕様。実機は文字サイズ2段階のため近い方に丸める(卓番40px以上で2倍角、品名22px以上で横2倍) |
| 予約注文画面から注文が流れない | MOCKモードで起動しているか `curl http://localhost:8000/api/health` の mode を確認 |
