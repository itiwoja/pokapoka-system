# 飲食店オーダリングシステム プロジェクト概要

**所属**: 沖縄みらいAI＆IT専門学校 2年生

---

## 1. プロジェクト発足の経緯

HitoFilm 様の知人である飲食店オーナーの方が、オーダーシステムを検討されており、その際、HitoFilm 様から「学生と協力して作ったらおもしろそうだね！」という提案をしてくださり、今回学校で引き受ける形となった。

- 村上壱基
- 小川優斗
- 野原一誠
- 平良隆之介

## 3. 各メリット

| 立場 | メリット |
|------|----------|
| 飲食店オーナー様 | 店舗のオーダリングシステムの基礎的な部分を費用を抑えて作成することができる |
| 学生 | 授業の内容よりも実際のシステム開発にさらに近いシステム開発の経験を積むことができる。ポートフォリオに書ける内容、面接でアピールできる内容が増える |

---

## 4. 役割

### 飲食店オーナー様

- 実現したいオーダリングシステムの構想を練っていただく
- 三幸学園からの納品物を元に、店舗で実際に使用するシステムの **開発・保守を担当してくれる企業の選定**
  - ※学校側で運用・サポートは不可なので、この部分は企業様にご担当いただく

### 学生

- 飲食店オーナー様への要望聞き取り
- 学校のカリキュラム等を考慮して **無理の無い開発範囲の決定**
- システムの **設計、開発、テスト、納品**

### 講師

- 学生の作業のサポート

## 8. システム概要（ハードウェア）

**クライアント・サーバ型システム** を現状では想定。

| 区分 | 想定構成 |
|------|----------|
| サーバ | 1台（DBサーバ、Webサーバ） |
| 注文用端末 | 複数台（タブレット端末を想定） |
| 注文確認用端末 | ディスプレイだけでサーバ機で兼用 or レシートプリンタ等を想定 |
| 支払い用端末 | 1台（POSレジを想定） |
