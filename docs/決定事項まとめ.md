# オーダーシステム 打合せ決定事項まとめ

> チーム内共有用、決定事項を抽出・整理したもの。
> ※ `打合せ決定事項.md` の **端末想定を Android（タブレット）に更新した最新版**。

---

## 1. プロジェクトの位置づけ

- 学校の **正課外プロジェクト** として進める
- 成果物の範囲: **設計書とソースファイル** までを納品
- 学校としての **保守運用は対象外**
- 運用保守企業の選定は **田島先生と相談のうえ決定**

---

## 2. 進行体制・スケジュール

| 項目 | 内容 |
|------|------|
| 作業時間 | 平日放課後・休日 |
| 進捗連絡 | チャット（Microsoft Teams）で随時 |
| 定例打合せ | 2週間に1度（オンライン） |
| 開催日時 | **毎月 第1・第3木曜日 18:00〜** |
| サポート | 裏で **現役エンジニアがサポート** |

---

## 3. 店舗の前提情報

### 3-1. 店舗規模

- 収容人数: **30名**
- ピーク時: **2〜3回転**
- 最大 **15組** 来店することもある

### 3-2. 席構成

| 席タイプ | 席数 |
|----------|------|
| 1人用 | 8席 |
| 2人用 | 3席 |
| 4人用 | 3席 |

### 3-3. スタッフ構成

- ネパール人 2名 + 日本人

### 3-4. 客層

- 外国人 : 日本人 ≒ **5 : 5**
- インバウンドが大半で **約8割が台湾人**
- その他: 香港・上海・シンガポール・韓国・中国・欧米

---

## 4. 導入の背景と目的

### 4-1. 課題

- 言語の壁 + 店舗規模の拡大
- ピーク時に一人ひとりへ丁寧な説明をする余裕がない
- 来店から **30分以内に提供したい** が、待たせてしまっている

### 4-2. 目指す姿

- スマートな店舗運営
- 回転率の向上
- 価格に見合った接客の維持
- 動画で食べ方などを分かりやすく案内

---

## 5. 開発スコープ

- **対象**: メニュー表示 〜 厨房へのオーダー転送まで
- **対象外**: 会計（既存の方法を継続）

---

## 6. 機能要件

### 6-1. 必須機能

- 新規客 / 予約客を **色分けで一目判別**
- **Sold Out 機能**
- **予約席の事前確保（席ロック）**
- **本日のおすすめ表示**（日次で簡単に編集可能）
- メニューの **新規登録・削除**
- **スクリーンセーバー機能**（動画3本程度をループ再生）
- **アレルギー表示**
- **時間経過の色 + 時間表示**

### 6-2. 多言語対応

- 日本語 / 英語 / 韓国語 / 中国語（簡体）/ 台湾語

### 6-3. メニュー表示項目

- アレルギー表記
- 動画再生
- オプション表示

### 6-4. 想定する将来拡張（要望段階）

- シフト管理のシステム化（現状は紙、全員分揃ってから印刷）
- 仕込みのデータ化・共有（言い忘れ防止・引き継ぎ保険）
- 仮テーブルの設定機能
- 予約客はタブレット操作不要にするフロー
- 厨房端末で完了押下 → 客側端末を **デザート催促画面に自動遷移**
- 厨房端末に **赤文字でバイト向けメモ** を追加できる機能

---

## 7. 厨房画面（最低限の表示項目）

- 新規 / 予約の区別
- オーダー受付時間
- 経過時間（何分経過したか）
- 人数
- 注文内容（略称で可）
- 卓番号
- オプションがある場合は **該当注文の直下に表示**

---

## 8. 業務フロー

### 8-1. 予約フロー

```text
予約システム
  ↓
オーダーシステムに転記
  ↓
予約確定（紙が出力 → 予約専用ボードに貼付）
  ↓
席をロック（新規客は割り当て不可）
```

**例**: 4名が 12:00〜予約 → 4人掛け席を **11:10〜ロック**

### 8-2. 来店〜提供フロー

```text
お客様着席
  ↓
スタッフが席に案内し、座席番号を入力後にタブレットを手渡し
  ↓
お客様がタブレットでオーダー
  ↓
厨房で調理
  ↓
提供
  ↓
厨房で完了操作 → 客側端末がデザート催促画面に遷移（将来拡張）
```

## 9. UI / UX 方針

- 新しくできる **公式Webサイトのデザインを基準** に統一
- 外観を保ち、**ブランディングを高く**
- 卓の呼び方は **「A1」形式**（慣れによりシステム側で別形式も可）
- **キッチン側UIに特にこだわる**
- お客様側UIも多少こだわる
