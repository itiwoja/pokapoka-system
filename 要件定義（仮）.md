## 2. プロジェクト概要

### 2.1 目的

* **スマートな店舗運営の実現**：言語の壁を解消し、少人数での効率的なオペレーションを確立する。
* **回転率の向上と提供遅延の解消**：ピーク時の提供時間（来店から30分以内）を遵守し、顧客を待たせない体制を作る。
* **価格に見合った接客の維持**：接客の質を落とさず、動画等で食べ方を分かりやすく案内する。

### 2.2 背景

* 店舗規模の拡大に伴い、言語の壁やピーク時のオペレーション負荷が増大している。
* 現在はスタッフ（ネパール人2名、日本人）に対し、客層の半分が外国人（うち8割が台湾人、その他アジア・欧米圏）であり、一人ひとりへ丁寧な説明をする余裕が不足している。

### 2.3 スコープ

* **対象業務：** メニュー表示 〜 厨房へのオーダー転送まで、および付随する座席管理機能（予約客対応、席ロック等）。
* **対象外：** 会計・決済処理（既存の方法をそのまま継続する）。学校としての保守運用。

## 3. 現状（As-Is）

* **現在の業務フロー：** 予約はシステム等から紙に出力し、専用ボードに貼付して管理。新規客も含め、スタッフが対面でメニュー説明・注文受付を行う。
* **使用しているツール・システム：** 電話での予約システム、紙の予約ボード、既存の会計システム、紙のシフト管理。
* **課題点：** 外国人客（全体の約5割）への説明に時間がかかる。注文受付の負荷が高く、注文から提供まで30分以上待たせてしまうことがある。
* **イレギュラー対応：** 言語の壁による聞き間違いや、ピーク時のオーダー混雑による提供遅延。
* **店舗規模（規模感）：** 収容人数30名（最大15組）、1人用：8席 / 2人用：3席 / 4人用：3席。ピーク時2〜3回転。
* **スタッフ構成：** ネパール人 2名 + 日本人

## 6. 機能要件

### 6.1 機能一覧

| No | 機能名 | 概要 | 優先度（MUST/SHOULD/COULD） |
| :--- | :--- | :--- | :--- |
| 1 | 【厨房】顧客属性判別表示 | 新規客 / 予約客を「色分け」によって一目で判別できるように表示する。 | MUST |
| 2 | 【注文】売切設定 | 残数がなくなったメニューを即座に売切状態にし、子機（タブレット）側で注文不可にする。 | MUST |
| 3 | 【予約】予約席の事前確保 | 予約客対応のため、対象の座席を新規客が割り当てられないようロックする。 | MUST |
| 4 | 【注文】本日のおすすめ表示 | 本日のおすすめメニューを分かりやすく表示する（日次で簡単に編集可能）。 | MUST |
| 5 | メニュー管理（管理） | メニューの新規登録・削除を簡単に実行できる。 | MUST |
| 6 | 【注文】スクリーンセーバー機能 | 待機中、食べ方案内などの動画（3本程度）をループ再生する。 | MUST |
| 7 | 【注文】アレルギー表示 | メニュー上にアレルギー情報を必ず表記・表示する（絶対必須）。 | MUST |
| 8 | 【厨房】時間経過表示 | 厨房端末でオーダー受付時間と、そこから何分経過したかを色と数値で表示する。 | MUST |
| 9 | 【注文】注文オプション | ごはん大盛や〇〇抜きなど商品のオプションを選択できる。 | SHOULD |

#### 6.1.2 多言語対応

* **対応言語：** 日本語 / 英語 / 韓国語 / 中国語（簡体） / 台湾語（繁体）

#### 6.1.3 メニュー表示項目

* アレルギー表記、動画再生、オプション表示

#### 6.1.4 想定する将来拡張（要望段階）

* シフト管理のシステム化（現状は紙、全員分揃ってから印刷）
* 仕込みのデータ化・共有（言い忘れ防止・引き継ぎ保険）
* 厨房端末で完了押下 → 客側端末をデザート催促画面に自動遷移
* 厨房端末に赤文字でバイト向けメモを追加できる機能

### 6.2 画面一覧

| No | 画面名 | 概要 | 備考 |
| :--- | :--- | :--- | :--- |
| 1 | 【注文】スクリーンセーバー | 待機中に動画3本程度をループ再生する画面。 | |
| 2 | 【注文】メニュー閲覧・注文 | 多言語、アレルギー、動画、オプションに対応した注文画面。 | |
| 3 | 【厨房】注文管理モニター | 新旧の注文状況、経過時間、顧客属性（色分け）を一覧表示する画面。 | キッチンUIに特にこだわる |
| 4 | 【親機】管理・メニュー編集 | 売切設定、本日のおすすめ、メニュー変更、席ロックを行う画面。 | |
| 5 | 【共通】卓番選択画面 | スタッフが個室などで端末を立ち上げる際、最初に表示される卓番設定画面。 | カウンターは常時起動 |

### 6.3 各機能の詳細要件

#### 6.3.1 タブレット運用方針

* **カウンター席：** 常時起動
* **個室：** 来店時にスタッフが立ち上げる（1組1台、団体は1テーブル1台）。初期画面は卓番を選択する画面（【共通】卓番選択画面）から開始する。
* **モニター表示仕様：** リモートでの確定が困難なため、**現場打合せで確認必須**。

#### 6.3.2 予約フォーム要件

オーナー要望（2026-05-17）に基づき、以下の入力項目と注意事項を持つ予約フォームを実装する。

##### 入力項目

1. 日時
2. 人数
3. 子供の有無（年齢を選択 または 備考欄に記載）
4. 名前
5. 連絡先（電話番号）
6. メールアドレス
7. メニュー予約の有無
8. アレルギーの有無（備考欄に記載）
9. その他備考欄（自由記入）

##### 注意事項（フォーム内に掲載）

* 大人の方 **1人1膳のご注文必須**（子供とのシェアは可）
* 提供まで **25〜30分** かかること
* 席の状況次第で、予約時間から **1組1時間半** のご利用になること

## 7. データ要件

### 7.1 データ項目一覧

#### 注文データ

| 項目名 | 型 | 必須 | 説明 |
| :--- | :--- | :--- | :--- |
| 注文ID | 文字列型 (Varchar) | ○ | 注文を一意に識別するID（自動採番） |
| 卓番号 | 文字列型 (Varchar) | ○ | 注文が発生した座席の番号（例：「A1」など） |
| 注文日時 | 日時型 (Datetime) | ○ | 注文が確定し、厨房に送信された日時 |
| 経過時間 | 数値型 (Integer) | ○ | 注文確定からの経過分数（アラート色変化の判定に使用） |
| 人数 | 数値型 (Integer) | ○ | 当該テーブルの着席人数 |
| 客層種別 | 文字列型 (Varchar) | ○ | 「新規」または「予約」の区別（厨房画面の色分けに使用） |
| メニューID | 文字列型 (Varchar) | ○ | 注文されたメニューの識別ID |
| オプションID | 文字列型 (Varchar) | － | トッピングやセット等の選択時のみ紐づくオプションID |

#### 予約フォームデータ

| 項目名 | 型 | 必須 | 説明 |
| :--- | :--- | :--- | :--- |
| 予約ID | 文字列型 (Varchar) | ○ | 予約を一意に識別するID（自動採番） |
| 日時 | 日時型 (Datetime) | ○ | 来店予定の日時 |
| 人数 | 数値型 (Integer) | ○ | 予約の総人数 |
| 子供有無（年齢） | 文字列型 (Varchar) | － | 子供同伴の有無、および選択された子供の年齢 |
| 氏名 | 文字列型 (Varchar) | ○ | 予約者の名前 |
| TEL | 文字列型 (Varchar) | ○ | 予約者の連絡先電話番号 |
| Mail | 文字列型 (Varchar) | ○ | 予約者のメールアドレス |
| 事前メニューの有無 | 真偽値型 (Boolean) | ○ | 来店前のメニュー予約があるかどうか（True/False） |
| アレルギーの有無 | 文字列型 (Varchar) | － | アレルギーがある場合の具体的な内容（備考欄入力値） |
| 備考 | 文字列型 (Text) | － | その他の要望などを自由記入する欄 |

#### メニューデータ

| 項目名 | 型 | 必須 | 説明 |
| :--- | :--- | :--- | :--- |
| メニューID | 文字列型 (Varchar) | ○ | メニューを一意に識別するID |
| メニュー名（5言語対応） | 文字列型 (Text/JSON) | ○ | 日本語/英語/韓国語/中国語(簡)/台湾語(繁)の各名称 |
| 価格 | 数値型 (Integer) | ○ | メニューの販売価格 |
| アレルギー情報 | 文字列型 (Varchar) | ○ | 含まれるアレルギー物質の表示テキスト（表示必須） |
| おすすめフラグ | 真偽値型 (Boolean) | ○ | 「本日のおすすめ」に表示するかどうか（True/False） |
| 売切フラグ | 真偽値型 (Boolean) | ○ | 「Sold Out」状態にするかどうか（True/False） |
| 動画URL | 文字列型 (Varchar) | － | メニュー詳細や食べ方案内動画の格納先URL |

### 7.2 マスタ一覧
